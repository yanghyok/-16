# 일정관리 대시보드 PRD

> 생성일: 2026-06-12
> 생성 방법: dashboard-prd skill (대화형)

## 1. 프로젝트 개요

### 배경 (Pain Points)
- 일정 관리를 엑셀로 운영 중이며, 지연 여부를 파악하려면 매번 파일을 직접 열어야 함
- 주간 회의 전 지연 업무를 사전에 정리하는 수작업 과정이 필요함
- 지연 여부는 확인되더라도 "왜 지연됐는지" 원인 데이터가 없어 논의가 비효율적

### 핵심 의사결정
- 지연 중인 업무를 즉시 식별하고, 원인을 파악하여 해결 조치(지원, 재배분, 일정 재조정 등)를 결정

### 목적
주간 회의에서 팀 전체가 지연 업무 현황을 한눈에 파악하고, 원인 기반 액션을 빠르게 결정한다.

### 핵심 가치
1. 엑셀을 열지 않아도 지연 현황을 즉시 파악
2. 지연 원인을 데이터로 관리하여 반복 패턴 발견 가능
3. 담당자별 현황 가시화로 업무 재배분 논의 효율화
4. 주간 회의 시간 단축 (현황 공유 → 액션 논의 중심으로 전환)

## 2. 데이터 소스

### 소스 1: 프로젝트진행상황.xlsx
- **위치**: `샘플/프로젝트진행상황.xlsx` → Google Sheets로 이전 권장
- **형태**: Excel (.xlsx) → Google Sheets (이전 후)
- **규모**: 5행 × 5컬럼 (초기 샘플 기준, 실운영 시 확장)
- **특이사항**: 부서 정보가 프로젝트명 앞 태그([영업], [구매], [정보])에 내장되어 있어 파싱 필요
- **필요 컬럼**: 담당자, 프로젝트명, 시작일, 종료일, 진행상황(%)
- **정제 규칙**:
  - 부서 추출: 프로젝트명의 `[태그]` 패턴에서 자동 파싱
  - 예상진행률 계산: `(오늘 - 시작일) / (종료일 - 시작일) × 100`
  - 지연 판단: `예상진행률 > 실제진행률`
  - 지연일수 계산: 마감 초과 업무는 `오늘 - 종료일`, 진행 중 지연은 `(예상진행률 - 실제진행률) / 100 × 전체기간`으로 환산

### 추가 필요 컬럼 (엑셀/Sheets에 추가)
- **지연사유**: 선택지 드롭다운
  - 승인 대기
  - 요건 변경
  - 리소스 부족
  - 외부 의존
  - 기술적 문제
  - 일정 재조정
  - 기타

## 3. 데이터 파이프라인

### 전체 흐름

```
[Google Sheets]
  담당자 / 프로젝트명 / 시작일 / 종료일 / 진행상황(%) / 지연사유
        |
        ▼
[Next.js API Route]
  - 부서 태그 파싱 ([영업] → "영업")
  - 예상진행률 계산
  - 지연 여부 판단
  - 지연일수 계산
        |
        ▼
[ISR 캐시] → [대시보드 UI]
```

### 통합 테이블 스키마

| 컬럼명 | 타입 | 소스 | 설명 |
|--------|------|------|------|
| 담당자 | string | Sheets 직접 입력 | 업무 담당자 이름 |
| 프로젝트명 | string | Sheets 직접 입력 | [부서태그] 포함 전체 이름 |
| 부서 | string | 프로젝트명에서 파싱 | [영업], [구매], [정보] 등 |
| 시작일 | date | Sheets 직접 입력 | 업무 시작일 |
| 종료일 | date | Sheets 직접 입력 | 원래 마감일 |
| 진행률 | number (%) | Sheets 직접 입력 | 실제 진행 퍼센트 |
| 예상진행률 | number (%) | 계산 | (오늘 - 시작일) / (종료일 - 시작일) × 100 |
| 지연여부 | boolean | 계산 | 예상진행률 > 실제진행률 |
| 지연일수 | number | 계산 | 마감 초과분(일) 또는 환산 지연 일수 |
| 지연사유 | string | Sheets 드롭다운 입력 | 7개 분류 중 선택 |

## 4. 대시보드 화면 (기능 정의 — 시각 위계는 디자인 툴 위임)

### View 1: 지연 현황 개요 — 전체 지연 상황을 5초 안에 파악하고 즉각 논의 업무를 식별

**KPI 항목** (우선순위 순):

| 순위 | 지표명 | 비교 기준 | 출처 | 계산 근거 |
|------|--------|-----------|------|-----------|
| 1 | 지연 업무 수 / 전체 대비 비율 | 전주 대비 증감 | Sheets | 예상진행률 > 실제진행률인 업무 COUNT |
| 2 | 마감 초과 업무 수 | - | Sheets | 종료일 < 오늘 AND 진행률 < 100% |
| 3 | 평균 지연 일수 | 전주 대비 증감 | Sheets | 지연 업무의 지연일수 평균 |

**표시하고 싶은 데이터 관계** (우선순위 순):

| 순위 | 데이터 관계 | 출처 |
|------|-------------|------|
| 1 | 지연 업무 목록: 담당자 · 프로젝트명 · 마감일 · 실제진행률 · 예상진행률 · 지연일수 · 지연사유 | Sheets |
| 2 | 지연 원인 분류별 업무 건수 분포 | Sheets (지연사유 컬럼) |

**필터**: 담당자 / 부서 / 지연 상태 (지연 중만 · 전체)

**데이터 투명성**:
- 예상진행률은 계산값(`~` 표시 권장)
- 지연사유 미입력 항목은 "미입력"으로 표시
- 데이터 기준 시점(갱신일)을 화면 상단에 표시

### AI 인사이트 제안 패널 — View 1

| 유형 | 탐지 조건 | 예시 문구 |
|------|-----------|-----------|
| 이상 탐지 | 전주 대비 지연 업무 수 +2건 이상 증가 | "지연 업무가 전주 대비 2건 증가했습니다. 원인 확인이 필요할 수 있습니다." |
| 추세 경고 | 특정 업무의 진행률이 2주 연속 변동 없음 | "[프로젝트명] 진행률이 2주간 변동 없습니다. 진행 가능 여부 확인을 권장합니다." |
| 기회 발견 | 마감 7일 이내 + 진행률 50% 미만 업무 존재 | "[프로젝트명] 마감 N일 이내이나 진행률이 낮습니다. 집중 지원을 검토해볼 수 있습니다." |

### View 2: 담당자별 현황 — 지연이 집중된 담당자를 파악하여 지원 또는 업무 재배분 논의

**KPI 항목** (우선순위 순):

| 순위 | 지표명 | 비교 기준 | 출처 | 계산 근거 |
|------|--------|-----------|------|-----------|
| 1 | 담당자별 지연 업무 수 | 전주 대비 | Sheets | 담당자 그룹핑 후 지연 업무 COUNT |
| 2 | 담당자별 평균 실제진행률 vs 예상진행률 | - | Sheets | 담당자별 두 값의 평균 비교 |

**표시하고 싶은 데이터 관계** (우선순위 순):

| 순위 | 데이터 관계 | 출처 |
|------|-------------|------|
| 1 | 담당자별 전체 업무 수 대비 지연 업무 수 비교 | Sheets |
| 2 | 담당자별 업무 상세 목록 (드릴다운: 특정 담당자 클릭 시 해당 업무 목록 표시) | Sheets |

**필터**: 담당자 / 부서 / 지연 상태

**데이터 투명성**:
- 담당자별 데이터는 현재 Sheets에 입력된 업무만 집계 (미입력 업무는 반영 안 됨)
- 데이터 기준 시점 표시

### AI 인사이트 제안 패널 — View 2

| 유형 | 탐지 조건 | 예시 문구 |
|------|-----------|-----------|
| 이상 탐지 | 특정 담당자의 지연 업무가 전체 지연의 50% 이상 | "[담당자]님 업무에 지연이 집중되어 있습니다. 지원 방안을 논의해볼 수 있습니다." |
| 추세 경고 | 담당자 전체 평균 진행률이 예상 대비 20%p 이상 하회 | "팀 전체 진행률이 예상보다 낮습니다. 일정 재조정이 필요할 수 있습니다." |
| 기회 발견 | 지연 없는 담당자의 여유 업무 슬롯 확인 가능 | "[담당자]님은 현재 지연 업무가 없습니다. 업무 재배분을 검토해볼 수 있습니다." |

### 데이터 신뢰성 및 제약

| 항목 | 내용 |
|------|------|
| 추정값 | 예상진행률은 기간 단순 비례 계산 (업무 복잡도 미반영) |
| 귀속 로직 | 지연 판단 기준: 예상진행률 > 실제진행률 |
| 갱신 주기 | 담당자가 Sheets에 직접 입력 시 반영 (주 1회 이상 입력 권장) |
| 알려진 제약 | 지연사유 미입력 시 원인 분석 불가. 초기 도입 시 입력 독려 필요 |

## 5. 기술 스택

| 기술 | 용도 | 선택 이유 |
|------|------|-----------|
| Next.js | 프론트엔드 프레임워크 | ISR 캐싱으로 주간 갱신 최적화 |
| Google Sheets | 데이터 소스 | 담당자가 브라우저에서 직접 입력·수정 가능, 별도 DB 불필요 |
| shadcn/ui | UI 컴포넌트 | 기본 slate 테마, 빠른 구현 |
| Recharts | 데이터 시각화 | React 친화적, 커스터마이징 용이 |
| Tailwind CSS | 스타일링 | 유틸리티 기반, 일관된 레이아웃 |
| TypeScript | 개발 언어 | 타입 안전성, 계산 로직 오류 예방 |

## 부록: 엑셀 → Google Sheets 이전 가이드

1. `프로젝트진행상황.xlsx`를 Google Drive에 업로드
2. Google Sheets로 열기 (자동 변환)
3. **지연사유** 컬럼 추가 후 데이터 유효성 검사(드롭다운)로 7개 선택지 설정
4. 시트 공유 설정: 팀원 편집 권한 부여
5. Next.js에서 Google Sheets API 연동 (gws CLI 또는 Service Account)
