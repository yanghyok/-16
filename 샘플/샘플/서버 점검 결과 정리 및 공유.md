# 서버 점검 결과 정리 및 공유

IBM i (AS/400) 서버 점검 메시지를 분석하고, 외부 업체에 발송할 메시지 초안을 작성하는 절차입니다.

---

## 분석 절차

### Step 1: 메시지 파싱

각 메시지에서 다음 항목 추출:

| 항목 | 필드명 |
|------|--------|
| 메시지 ID | Message ID |
| 심각도 | Severity (0~99) |
| 메시지 유형 | Message type |
| 발생 일시 | Date sent + Time sent |
| 메시지 내용 | Message |
| 원인 | Cause |
| 복구 방법 | Recovery |

---

### Step 2: 상태 분류 기준

| 기준 | 분류 |
|------|------|
| Severity 99 + Inquiry | 즉시 처리 필요 (단, Step 3 교차 분석 선행) |
| Severity 60~98 | 경고 — 확인 필요 |
| Severity 00~59 Completion/Information | 정상 |
| Severity 00 순수 정보성 | 정상 — 업체 발송 불필요 |

---

### Step 3: 교차 분석 (필수)

메시지 간 연관성을 반드시 확인한 후 분류 확정:

- **Completion 메시지(CPF1164 등) end code 0 확인**: 이전 Inquiry/오류 메시지가 있어도 Job 정상 완료로 판정 → 긴급도 하향, "(참고용)" 처리
- **타이밍 이상 확인**: Job 종료 후 수 시간 뒤 해당 Job 관련 경고 발생 시 → 잔여 잠금 등 이상 징후로 판정, 업체 확인 요청
- **시각 오기재 방지**: 업체 발송 시 각 메시지의 정확한 시각을 원본과 재대조

---

### Step 4: 발송 대상 선별

다음 메시지는 업체 발송 **생략**:

| 메시지 유형 | 예시 |
|------------|------|
| Severity 00 순수 정보성 | CPF7019 (시퀀스 번호 안내) |
| 일반 Job 기동 | CPF1124 |
| Job 정상 종료 (end code 0) | CPF1164 |
| 사인온 처리 | CPIAD0B |
| 스케줄 Job 정상 제출 | CPC1236 |

---

### Step 5: 업체 발송 메시지 초안 형식

```
안녕하세요.

[날짜] 서버 점검 중 확인된 메시지 [N]건에 대해
검토 요청 드립니다.

──────────────────────────────────────

[번호]. [긴급도] [내용 한줄 요약]

   일시: YYYY-MM-DD HH:MM:SS
   메시지: [Message 본문] ([Message ID], Severity [N])
   내용: [발생 상황 한국어 설명]
         [교차 분석 결과 포함 — 예: "이후 HH:MM에 정상 완료 확인됨"]
   요청: [업체에 구체적으로 요청할 사항]

──────────────────────────────────────

확인 후 결과 공유해 주시면 감사하겠습니다.

감사합니다.
```

---

## 긴급도 표기 기준

| Severity | 상황 | 긴급도 표기 |
|----------|------|------------|
| 99 | Job 완료 미확인 또는 미응답 | [긴급] |
| 99 | 후속 Completion end code 0 확인됨 | (참고용) |
| 60~98 | 확인 필요 | (확인 필요) |

---

## 요청 문구 예시

| 상황 | 요청 문구 |
|------|----------|
| 테이프 볼륨 소진 | 백업 데이터 정상 완료 여부 및 테이프 교체 처리 방식 확인 부탁드립니다. |
| 대량 잠금 경고 | 잔여 잠금 여부 및 서버 영향도 확인 부탁드립니다. |
| Job 비정상 종료 | Job 재실행 필요 여부 및 원인 확인 부탁드립니다. |
| 타이밍 이상 경고 | 해당 시각 기준 이상 징후 여부 및 서버 영향도 확인 부탁드립니다. |
| 시퀀스 번호 경고 | 서버 영향이 없는지 확인 부탁드립니다. |

## 참고: 주요 IBM i 메시지 ID

| Message ID | 의미 | 일반적 처리 |
|------------|------|------------|
| CPA4088 | 테이프 볼륨 소진 (응답 대기) | Severity 99, 완료 여부 교차 확인 |
| CPF1164 | Job 종료 | end code 0이면 정상 완료 |
| CPF1124 | Job 기동 | 정보성, 발송 불필요 |
| CPI1466 | 대량 잠금 경고 | Severity 60, 타이밍 이상 여부 확인 |
| CPF7019 | 저널 시퀀스 번호 안내 | Severity 00, 발송 불필요 |
| CPC1236 | 스케줄 Job 제출 완료 | 정보성, 발송 불필요 |
| CPIAD0B | 사인온 서버 처리 | 정보성, 발송 불필요 |

## 주의사항

- 각 메시지의 **정확한 시각** 반드시 원본과 대조 후 기재 (오기재 방지)
- Severity 00 정보성 메시지는 업체 발송 생략 (불필요한 확인 요청 방지)
- 교차 분석 결과(완료 메시지, 타이밍 이상 등)를 발송 메시지에 반영
- 발송 전 수신자(업체명/담당자명)와 발신자 서명 추가
