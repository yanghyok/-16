1. 구매 발주/입고 시스템 개선 요청의 건

- 요청 목적: 프로젝트 회의시 시스템 개선 하라는 회장님 지시사항(다음주 수요일 보고예정)

- 요청 사항:
   현재 발주/입고 시스템 상으로는 예를 들어
  ex)  품번 : RB1730   수량 : 500ea   납기: 10월 20일 발주가 나갔을 경우

        10월에 납품이 진행 되서 10월1일~20일사이  입고가 됨,
        그런데 만일 업체에서 미리 생산이 완료되서 9월 20일에 납품이 될 경우에도
        입고가 가능한 구조임.
   최근 프로젝트 진행 과정에서 
   몇개 업체에 년간 사용예상 수량을 필요한 납기로 일괄 선 발주 진행 하면서 
   하기 지시 사항 발생.

    ※ 지시 사항
    : 10월납기인데 미리 제품을 9월이나 8월로 당겨서 입고가 안되게 담당자가 관리할 것이 
      아니라 전산으로 입고가 안되게 전산 시스템으로 개선하라는 지시.

   ◈ 요청 내용 
     ex)  품번 : RB1730   수량 : 500ea   납기: 10월 20일 발주가 나갔을 경우
           10월 이전에는 입고가 불가 하도록 전산 시스템 개선 요청드림.

 - 기대 효과:
   : 발주품이 필요한 시점이 아닌 사전에 입고 될 경우 업체에 대금이 미리 나가는 문제 방지
     (금융손실 비용 발생 방지)

2. 현장특가관리 개선
 
- 문제점
	- 현장코드 등록 후 진행 상황 관리 안함
          수기 관리하여 문제 발생
- 개선점
	- 전산 진행 상황에 따라 진행 상황 자동 적용
		-> 승인특가 등록 후 승인 시 현장코드 확정 상태로 변경
		-> 특가 주문 등록 하여 출고 시 현장코드 진행으로 변경
		-> 특가 출고 90%이상이면 완료 상태로 변경
		-> 납기 예정 지나면 OUT 상태로 변경
- 목표
	- 특가금액, 특가미출고수량, 출고실적, 수주 잔량 등 조회
          수주 대 OUT 비율 등 관리
- 연동
	- 현장특가관리코드 - 승인특가 현장코드
	- 승인특가 작성일/번호/라인 - 주문입력 특가작성일/번호/라인
- 일정
	- 영업 지원에서 정리 후 일정 정리 예정


3. 현장 영업진행 관리 프로그램 개발									

 - 전산 개발 최종 목표 : 현장 등록을 자동 견적을 통해 진행해야만, 최종적으로 승인특가 작성 가능									
    → 자동 견적 미 진행 시, 현장 등록은 가능하나, 별도 승인특가 적용 불가									
 - 현싯점 개발 요청사항									
   1) 현장코드가 있어야, 승인특가 작성 가능									
     - 승인특가 작성 시, 현장관련 사항을 기입하여 작성하면 현장관리 프로그램에 현장이 자동 등록되도록 개발 진행									
     - 또한, 현장관리 프로그램에서도 현장별 승인특가 작성 여부, 특가금액 및 출고금액이 확인되도록 개발 진행 									
   2) SPEC OUT시 대체출고 메이커 및 대체사유를 기입하여야 확정 가능하도록 개발
									
   3) 현장코드는 담당자별로 작성순서에 따라, 자동 부과									
   4) 현장관리 프로그램에 다량한 써칭기능 부여 : 영업지원 별도 필요한 써칭기능 통보									
      - ① 기간별 현장 현장, ② 기간별 현장 출고내역, 금액, ③ 기간별 SPEC OUT현장 내역, ④  기타									
   5) 관공서 영업PJ회의 보고양식 프로그램 개발
