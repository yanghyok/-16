======================================================
  한양혁 AX 제안서 — 쉬운 설명
======================================================

■ 이 문서는 무엇인가요?
"한양혁 님의 업무에 AI(Claude Code)를 도입하는 단계별 실행 계획서"입니다.

======================================================
■ 핵심 요약
======================================================

현재 두 가지 주요 업무 부담이 있습니다.

  [업무 1] 서버·백업 점검
  - 문제점: 에러 원인 파악과 외부 업체 커뮤니케이션이 시간·부담이 크다

  [업무 2] 개발 요청서 분석
  - 문제점: 요청이 모호해서 인터뷰를 여러 번 반복해야 한다

두 업무 공통점:
  "뒤죽박죽 텍스트 → 깔끔한 보고·공유 산출물"로 만드는 작업

======================================================
■ 어떻게 AI를 도입하나요?
======================================================

"스킬(자동화 루틴)을 만드는 과정"을 단계별로 안내합니다.

처음부터 완성된 시스템을 만들려 하지 말고,
작은 것부터 써보고 → 확인하고 → 다듬는 방식으로 진행합니다.

  의도 전달 → 결과 확인 → 조정 → 만족 → 스킬화 → 반복 사용 → 업데이트

======================================================
■ 스킬 1. 서버 점검 결과 정리 (8단계)
======================================================

  1단계: 에러 메시지 붙여넣기 → 정상/경고/즉시 처리 필요 분류
  2단계: 전체 점검 항목 상태 분류
  3단계: 이상 항목만 필터링
  4단계: 원인 후보 제안
  5단계: 외부 업체 문의 초안 작성
  6단계: 팀원 공유 메시지 자동 생성
  7단계: 형식 다듬기
  8단계: 이 흐름을 "스킬"로 저장

  ※ 각 단계 후 사람이 직접 확인하는 과정이 필수입니다.

======================================================
■ 스킬 2. 개발 요청 요구사항 구체화 (8단계)
======================================================

  1단계: 요청서 붙여넣기 → 불명확한 부분 지적
  2단계: 인터뷰 질문 목록 생성
  3단계: 인터뷰 메모 반영
  4단계: 핵심 기능 / 부가 기능 분류
  5단계: 최종 요구사항 정리
  6단계: 팀장·임원 보고용 1장 요약
  7단계: 형식 다듬기
  8단계: 이 흐름을 "스킬"로 저장

======================================================
■ 완성 후 모습
======================================================

  에러 메시지 붙여넣기  →  [서버 점검 스킬]  →  팀원 공유 메시지 자동 완성
  개발 요청서 붙여넣기  →  [요구사항 스킬]   →  보고용 요약 자동 완성

======================================================

   AI와 함께 단계별로 자동화 루틴으로 만들어 나가는 실행 로드맵"
