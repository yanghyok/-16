# 전산개발 시간 단축 — 탐색 및 수주잔량 설계 초안

_작성일: 2026-05-13 | thinking-partner 세션_

---

## 핵심 인사이트

**병목 구조:**
```
현업이 말로 요청
    ↓
개발자가 직접 DB 구조 + 화면 구성을 추론·설계  ← 병목 (수일 소요)
    ↓
코딩 시작
```

**문제의 본질**: 요청이 많고 매번 새로운 성격 → 기존 선례 재활용 불가 → 개발자가 백지에서 설계 시작

**AI 활용 개선안:**
```
현업이 말로 요청
    ↓
AI가 요구사항 문서화 + DB 구조 + 화면 구성 초안 생성 (30분)
    ↓
개발자 검토·확정 (반나절)
    ↓
코딩 시작
```

개발자가 **설계자 → 검토자**로 역할 전환이 핵심.

## 실습: 수주잔량 조회 화면

### 요구사항

- **사용자**: 영업팀
- **목적**: 담당 거래처의 미출하 수주를 확인하고 납기 대응
- **필요 정보**: 거래처, 품목, 수주일, 납기일, 수주수량, 출하수량, 잔량, 잔액, 담당 영업사원

---

### DB 구조 초안

```sql
-- 수주 헤더
orders
  order_id       PK
  customer_id    FK → customers
  rep_id         FK → sales_reps
  order_date     DATE
  status         ('확정','취소')

-- 수주 라인 (품목별)
order_items
  item_id        PK
  order_id       FK → orders
  product_id     FK → products
  qty_ordered    INT        -- 수주수량
  qty_shipped    INT        -- 출하수량
  qty_remaining  COMPUTED   -- 잔량 (= qty_ordered - qty_shipped)
  unit_price     DECIMAL
  due_date       DATE       -- 납기일

-- 품목
products
  product_id     PK
  product_code   VARCHAR
  product_name   VARCHAR
  category       VARCHAR    -- 도기/비데/수전 등
```

> 기존 `customers` 테이블 그대로 활용 가능
