# 개발 요청 요구사항 구체화 스킬

모호한 개발 요청서를 단계별로 구체화하여 인터뷰 횟수를 줄이고 보고용 산출물까지 자동 생성하는 스킬.

## 트리거 키워드

"개발 요청", "요구사항 정리", "요청서 분석", "기능 정의", "인터뷰 질문 만들어줘",  
"보고용 요약", "개발 요청서 붙여넣을게", "요청 내용 구체화"

---

## 실행 흐름

```
요청서 붙여넣기
  → [1단계] 불명확 항목 탐지
  → [2단계] 인터뷰 질문 목록 생성
  → (인터뷰 진행 — 사용자)
  → [3단계] 인터뷰 답변 반영
  → [4단계] 핵심/부가/추후 분류 제안
  → [5단계] 최종 요구사항 확정
  → [6단계] 보고용 1장 요약 생성
```

각 단계 후 사용자 확인 필수. 한 번에 다 하지 않고 단계별로 진행.

---

## 단계별 처리 방법

### 1단계: 불명확 항목 탐지

요청서를 받으면 아래 항목을 기준으로 불명확한 부분을 찾아 목록으로 제시:

**탐지 기준:**
- **범위 불명확**: "관련 데이터 조회" — 어떤 데이터? 어느 범위?
- **조건 누락**: "특정 조건에서 알림" — 어떤 조건? 누구에게?
- **화면/UI 불명확**: "목록으로 보여줘" — 어떤 컬럼? 정렬 기준?
- **예외 처리 누락**: 오류 시 어떻게? 권한 없는 사용자는?
- **연동 대상 불명확**: 어떤 시스템과? 실시간/배치?
- **완료 기준 없음**: 어떻게 되면 개발 완료인가?

출력 형식:
```
📋 불명확 항목 (N개)

1. [항목명] — "[원문 발췌]"
   → 불명확한 이유: ...

2. ...
```

### 2단계: 인터뷰 질문 목록 생성

불명확 항목을 해소할 질문을 **업무 흐름 순서**대로 정렬하여 제시.

질문 작성 원칙:
- 요청자가 "예/아니오" 또는 구체적 답변이 가능한 수준으로 작성
- IT 전문 용어 지양 — 요청 부서 담당자가 이해할 수 있는 표현
- 한 질문에 하나의 주제만

출력 형식:
```
🗣️ 인터뷰 질문 목록 (N개)

[업무 흐름: 입력 → 처리 → 출력 순]

1. (불명확 항목 해소) 질문 내용?
2. ...

※ 인터뷰 후 답변을 붙여넣어 주시면 반영하겠습니다.
```

### 3단계: 인터뷰 답변 반영

사용자가 인터뷰 메모를 붙여넣으면:
1. 각 질문 항목에 답변 매핑
2. 아직 답변되지 않은 항목 표시
3. 구체화된 요구사항 목록 갱신

출력 형식:
```
✅ 답변 반영 결과

[반영됨] 질문1 → 확정 내용: ...
[반영됨] 질문2 → 확정 내용: ...
[미답변] 질문3 → 추가 확인 필요

📋 현재 확정된 요구사항 (N개)
1. ...
```

### 4단계: 핵심/부가/추후 분류 제안

확정된 요구사항을 3가지로 분류 제안:

| 구분 | 기준 |
|------|------|
| **핵심 기능** | 이것 없으면 요청 목적 달성 불가 |
| **부가 기능** | 있으면 좋지만 없어도 운영 가능 |
| **추후 검토** | 현 단계에서 범위·리소스 초과, 2차 개발 대상 |

출력 후 사용자 조정 요청:
```
📊 기능 분류 제안

🔴 핵심 기능 (N개)
  - ...

🟡 부가 기능 (N개)
  - ...

⬜ 추후 검토 (N개)
  - ...

조정할 항목이 있으면 말씀해 주세요.
```

### 5단계: 최종 요구사항 목록 확정

사용자 조정 반영 후 최종 목록 정리:
- 각 항목에 "요청자 확인 완료 여부" 표시
- 핵심 기능에 한해 간단한 검수 기준 추가

### 6단계: 보고용 1장 요약 생성

팀장·임원 보고에 쓸 수 있는 형식으로 자동 생성:

```
■ 개발 요청 요약

요청 부서: [부서명]          요청일: YYYY-MM-DD
요청자:   [담당자]          접수번호: (있는 경우)

■ 요청 배경
[왜 필요한지 2~3줄]

■ 핵심 기능 (N개)
1. ...   [요청자 확인: ✅/미확인]
2. ...

■ 부가 기능 (N개)
1. ...

■ 추후 검토 항목
1. ...

■ 예상 작업 범위
• 영향 시스템: ...
• 예상 일정: (미정 시 "협의 필요" 기재)
• 특이사항: ...
```

---

## 운영 규칙

- **한 단계씩 진행**: 모든 단계를 한 번에 출력하지 않음. 사용자가 확인 후 다음 단계로
- **원문 보존**: 요청서 원문을 수정하지 않음. 해석만 추가
- **모호함 잔존 허용**: 인터뷰 후에도 불명확한 항목은 "(확인 필요)"로 표기하고 진행
- **분류는 제안**: 핵심/부가/추후 분류는 AI 제안이며 최종 결정은 사용자
- **보고 요약은 초안**: 예상 작업 범위·일정은 사용자가 직접 보완

---

## 스킬 업데이트 방법

실무에서 자주 나오는 불명확 패턴 발견 시:
> "이런 요청은 항상 A를 먼저 확인해야 해. 스킬에 추가해줘."

→ 탐지 기준 목록 또는 질문 작성 원칙에 추가.
