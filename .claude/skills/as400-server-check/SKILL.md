# AS/400 서버 점검 스킬

AS/400 시스템 메시지 로그를 분류하고 팀 공유 메시지를 자동 생성하는 스킬.

## 트리거 키워드

"서버 점검", "AS400 로그", "에러 분류", "점검 결과 공유", "서버 이상",  
로그 텍스트에 `Message ID`, `CPA`, `CPF`, `CPI`, `Severity` 등이 포함되면 자동 실행.

---

## 분류 기준

| 등급 | 조건 | 표시 | 의미 |
|------|------|------|------|
| 즉시 처리 필요 | Message type = Inquiry (CPA) **또는** Severity 70 이상 | 🔴 | 응답 대기 중이거나 심각한 오류. 당일 처리 필수 |
| 경고/모니터링 | Severity 40~69 (Information/Diagnostic) | 🟡 | 이상 징후. 확인 후 판단 필요 |
| 정상 범위 | Severity 0~39 (Information) | 🟢 | 운영 중 발생하는 일반 메시지 |

### AS/400 메시지 ID 접두어 참고
- **CPA** — Action Required (Inquiry). Severity와 무관하게 항상 🔴
- **CPF** — 시스템 오류. Severity 기준 적용
- **CPI** — 정보성 메시지. Severity 기준 적용
- **MCH** — 기계 검사(Machine Check). 항상 🔴
- **TAP/SAV** — 테이프/백업 관련. Severity 기준 적용

---

## 실행 흐름

### 1단계: 로그 수신
사용자가 AS/400 메시지 텍스트를 붙여넣으면 자동 파싱.

파싱 대상 필드:
- `Message ID` — 메시지 코드
- `Severity` — 심각도 숫자
- `Message type` — Inquiry / Information / Diagnostic / Completion
- `Date sent` / `Time sent` — 발생 시각
- `Message` — 메시지 본문 요약
- `Cause` — 원인 설명
- `Recovery` — 복구 절차

### 2단계: 분류 및 정리

아래 형식으로 분류 결과 출력:

```
📋 서버 점검 결과 — YYYY-MM-DD

🔴 즉시 처리 필요 (N건)
  ├ [메시지ID] Severity XX | HH:MM | 한 줄 요약
  └ ...

🟡 경고/모니터링 (N건)
  ├ [메시지ID] Severity XX | HH:MM | 한 줄 요약
  └ ...

🟢 정상 범위 (N건)
  └ [메시지ID] Severity XX | HH:MM | 한 줄 요약
```

### 3단계: 즉시 처리 항목 원인 후보 제안

🔴 항목이 1건 이상이면 각 항목에 대해:
- 원인 후보 2~3가지 (AS/400 메시지 ID 기반)
- 각 후보별 확인 방법 1줄 (AS/400 명령어 포함)
- 외부 업체 문의 필요 여부 판단

### 4단계: 팀 공유 메시지 초안 생성

아래 형식으로 공유 메시지 자동 생성:

```
[서버 점검 결과] YYYY-MM-DD

■ 요약
• 🔴 즉시 처리: N건
• 🟡 확인 필요: N건  
• 🟢 정상: N건

■ 즉시 처리 항목
• [메시지ID] 내용 요약 → 조치 방향

■ 처리 결과 / 미처리 항목
• (직접 입력 필요)

— 정보관리팀 양혁
```

### 5단계: 발송 확인 (선택)

분류 결과와 공유 메시지 초안을 보여준 뒤,  
**"Teams로 보내줘" / "메일로 보내줘"** 라고 하면 MS365 MCP로 즉시 발송.

발송 전 반드시 확인:
- 수신자/채널 확인
- 메시지 내용 최종 검토

## 외부 업체 문의 초안

🔴 항목 중 직접 처리가 어려운 경우 문의 초안 자동 생성:

```
제목: [AS/400] [메시지ID] 관련 문의

안녕하세요.

아래와 같은 오류가 발생하여 문의드립니다.

• 발생 일시: YYYY-MM-DD HH:MM
• 메시지 ID: [메시지ID] (Severity [숫자])
• 현상: [Message 내용]
• 원인(추정): [Cause 내용]

확인 및 조치 방법을 안내해 주시면 감사하겠습니다.

로얄앤컴퍼니 정보관리팀 양혁
```

---

## 운영 규칙

- 분류 결과를 먼저 보여주고, 사용자 확인 후 공유 메시지 생성
- 불확실한 분류는 🟡 경고로 처리하고 "(확인 필요)" 표기
- Severity 0이어도 CPA(Inquiry) 타입이면 반드시 🔴 처리
- 공유 메시지는 초안이므로 "처리 결과/미처리 항목"은 사용자가 직접 입력
- 새로운 메시지 ID 패턴이 나오면 사용자에게 분류 기준 확인 후 이 스킬 업데이트

---

## 스킬 업데이트 방법

실무에서 새 패턴 발견 시:
> "이 메시지는 경고가 아니라 즉시 처리야. 스킬에 반영해줘."

→ 분류 기준 테이블 또는 메시지 ID 접두어 참고 목록에 추가.
